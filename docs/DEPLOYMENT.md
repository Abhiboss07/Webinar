# Deployment & Operations

The platform has three deployable parts:

| Part | What | Where (recommended) |
|---|---|---|
| **backend/** | Node/Express API + CMS + workers | Render Web Service (or Docker) |
| **admin/** | React/Vite admin panel (static) | Render Static / Netlify / Vercel / nginx |
| **frontend/** | Public workshop site (static) | Hostinger (or any static host / CDN) |

`MONGODB_URI` + `JWT_SECRET` must be real env before boot; **everything else the client manages from
Settings** (Razorpay, Cloudinary, SMTP, Google Sheets, branding, SEO…).

---

## 1. MongoDB (Atlas)
Create a free cluster, a DB user, allow network access `0.0.0.0/0` (Render egress is dynamic), copy the
`mongodb+srv://…` string.

## 2. Backend → Render
- New **Web Service**, root `backend/`, build `npm install`, start `npm start` (runs `validate:env` first).
- **Environment** (dashboard): `MONGODB_URI`, `JWT_SECRET` (`openssl rand -hex 32`), `SETTINGS_ENC_KEY`
  (another random 32-byte hex), `RAZORPAY_KEY_ID/SECRET`, `GOOGLE_SHEET_ENDPOINT`, `SHEET_SHARED_TOKEN`,
  `ADMIN_EMAIL`, `ADMIN_PASSWORD`, `FRONTEND_URL` (public site + admin origins, comma-separated),
  `API_URL` (this service's URL), `NODE_ENV=production`. Optional: `STORAGE_PROVIDER=cloudinary` +
  `CLOUDINARY_*`, `COMM_WORKER_MS`, `BACKUP_INTERVAL_HOURS=24`, `API_RATE_MAX`.
- Health check path: `/health` (readiness: `/health/ready`).
- One-time via Render Shell:
  `npm run seed:admin && npm run migrate:config && npm run seed:workshop && npm run seed:templates`.

## 3. Admin panel → Render Static (or Netlify/Vercel)
- Build `npm install && npm run build`, publish `admin/dist`, root `admin/`.
- Env: `VITE_API_BASE=https://<render-backend>`, `VITE_PUBLIC_SITE_URL=https://<public-site>`.
- SPA rewrite: all paths → `/index.html`. Add the admin origin to backend `FRONTEND_URL`.

## 4. Public site → Hostinger
- `cd frontend && npm run build:css`, then upload the **static files from `frontend/`** (`assets/`, `css/`,
  `js/`, `sections/`, `config/`, `index.html`, `thank-you.html`, `.htaccess`) — omit `src/`, `node_modules/`,
  `package.json`, `tailwind.config.js`.
- Set `api.prod` in `config/workshop-config.js` to the Render URL (used to resolve the API + offline
  fallback). `robots.txt` / `sitemap.xml` are generated by the API — point the domain/CDN at
  `<API_URL>/robots.txt` and `/sitemap.xml`, or copy their output to the static host.

## Docker (self-host all three)
```bash
cp backend/.env.example backend/.env   # fill MONGODB_URI (mongodb://mongo:27017/youngness), JWT_SECRET, …
docker compose up --build
# API :4000 · admin :8080 · mongo :27017
docker compose exec backend npm run seed:admin
docker compose exec backend sh -c "npm run migrate:config && npm run seed:workshop && npm run seed:templates"
```

## Operations
- **Health:** `/health` (liveness), `/health/ready` (DB). **Backups:** Admin → System → Backups (manual)
  or set `BACKUP_INTERVAL_HOURS` for automatic config backups (keeps last 10). Restore is Super-Admin-only.
- **Maintenance mode:** Admin → System → Maintenance (public site shows a message; `?preview=1` bypasses).
- **Monitoring:** Admin → System (health lights, queue, logs, storage, security, notifications) and
  Analytics for business metrics.
- **Security:** Helmet, gzip/brotli compression, global + per-route rate limiting, query sanitization,
  bcrypt, JWT + refresh tokens, RBAC, account lockout, encrypted settings secrets, full audit log.

## CI
`.github/workflows/ci.yml` validates backend env, smoke-requires the app, and builds the admin on every
push/PR.
